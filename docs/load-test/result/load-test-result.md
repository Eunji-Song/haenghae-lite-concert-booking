# 콘서트 좌석 예약 시스템 부하 테스트 보고서

## 1. 테스트 개요

### 1.1 테스트 목적
본 테스트는 콘서트 예약 서비스에서 동일 좌석 예약 요청이 동시에 발생하는 상황을 가정하여,
시스템의 동시성 처리 안정성과 예외 처리 적절성을 검증하기 위해 수행하였다.

특히 분산 환경에서의 중복 예약 방지, 락 처리, 트랜잭션 안정성을 중점적으로 확인하는 것을 목표로 한다.

---

## 2. 테스트 대상 선정 이유

콘서트 예약 시스템에서 좌석 예약 기능은 다음과 같은 특성을 가진다.

- 다수의 사용자가 동시에 접근하는 고경합 구간
- 중복 예약 발생 시 서비스 신뢰도에 치명적인 영향
- 분산 락, 트랜잭션, 예외 처리 로직이 집중되는 핵심 기능

따라서 본 테스트에서는 가장 부하와 위험도가 높은 동일 좌석 예약 API를 테스트 대상으로 선정하였다.

---

## 3. 테스트 시나리오

### 3.1 사용자 흐름
1. 사용자 로그인
2. 대기열 토큰 발급
3. 좌석 예약 요청

### 3.2 테스트 조건
- 모든 사용자는 동일한 콘서트 ID, 날짜, 좌석 번호로 예약 요청
- 이미 예약된 좌석에 대해서는 409 Conflict 응답을 정상으로 간주
- 그 외 상태 코드는 비정상 응답으로 분류

---

## 4. 테스트 환경

- 부하 테스트 도구: k6
- 실행 환경: Local
- 테스트 방식: ramping-vus
- 최대 동시 사용자 수: 200

---

## 5. 테스트 결과 요약

### 5.1 주요 결과
- 대부분의 요청이 409 Conflict(이미 예약된 좌석)으로 정상 처리됨
- 일부 요청에서 500 Internal Server Error 발생
- p95 응답 시간이 기준을 초과하여 threshold 실패
